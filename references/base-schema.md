# 飞书 Base 结构

将这份结构用于归档表。默认新建的 Base 文档名应为 `小红书笔记采集`。

## 规范表定义

- 默认 Base 文档名：`小红书笔记采集`
- 默认数据表名：`小红书笔记采集`
- 每条小红书笔记对应一行
- 主匹配键：`内容链接`
- 如果用户后续自己调整字段顺序，不再主动干预

## 必需字段

| 字段 | 类型 | 必需 | 来源 | 说明 |
|------|------|------|------|------|
| `标题` | `text` | 是 | `title` | 笔记标题 |
| `内容类型` | `text` | 是 | `note_type` | 图文 / 图集 / 视频 |
| `正文` | `text` | 否 | `content` | 笔记正文 / 描述 |
| `标签` | `select` | 否 | `tags` | 多选字段；每个标签是一个选项 |
| `发布日期` | `datetime` | 否 | `published_at` | 用飞书支持的分钟级时间格式写入 |
| `点赞` | `number` | 否 | `like_count` | 整数，显示为 0 位小数 |
| `收藏` | `number` | 否 | `collect_count` | 整数，显示为 0 位小数 |
| `评论` | `number` | 否 | `comment_count` | 整数，显示为 0 位小数 |
| `内容链接` | `url` | 是 | `note_url` | 规范化后的小红书笔记链接 |
| `作者主页链接` | `url` | 否 | `author_url` | 小红书作者主页链接 |
| `图片链接` | `text` | 否 | `image_urls` | 以换行拼接的远程 URL |
| `图片附件` | `attachment` | 否 | 本地图片文件 | 在一次 Skill 执行里自动下载并上传 |
| `视频链接` | `text` | 否 | `video_urls` | 以换行拼接的远程 URL |
| `视频附件` | `attachment` | 否 | 本地视频文件 | 在一次 Skill 执行里自动下载并上传 |
| `采集时间` | `datetime` | 否 | 运行时写入 | 本次同步执行时间 |

## 默认阅读顺序

默认视图的字段顺序应设置为：

1. `标题`
2. `内容类型`
3. `正文`
4. `标签`
5. `发布日期`
6. `点赞`
7. `收藏`
8. `评论`
9. `内容链接`
10. `作者主页链接`
11. `图片链接`
12. `图片附件`
13. `视频链接`
14. `视频附件`
15. `采集时间`

## 推荐记录映射

仅使用已确认的值构造记录载荷。

`lark-cli base +record-upsert` 示例 JSON：

```json
{
  "标题": "示例笔记",
  "内容类型": "图文",
  "正文": "示例正文",
```

## 结构修复规则

- 补齐缺失字段
- 除非必要，不要重命名用户已有字段
- 如果规范字段已存在但类型不兼容，仅在无法安全直接更新时创建安全替代字段
- 仅在用户明确要求时删除冗余字段
- 当前约定下，`单选`、`采集结果`、`失败原因` 属于应删除的冗余字段
- 若 `attachment` 字段受接口限制无法直接改名，允许创建目标字段并在默认视图中隐藏旧字段

## 附件策略

- `图片附件` 和 `视频附件` 必须是 `attachment` 字段
- 远程 URL 应写入 `图片链接` 和 `视频链接`
- 附件字段只能上传本地文件
- 对用户而言，附件上传应是一次 Skill 执行内的自动流程，而不是额外拆成独立人工步骤
